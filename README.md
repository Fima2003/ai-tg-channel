# Daily Telegram Image Bot

A Python Telegram bot built with Telethon. It posts a daily batch of 1 to 4 generated images to a channel along with short text generated by a local small LLM.

The first version uses a placeholder image generator built on Pillow, so the full flow is ready now and the image backend can be swapped later.

## Requirements

- Python 3.10 or newer
- Conda environment: `tg-of-ai`
- Telegram bot credentials from [my.telegram.org](https://my.telegram.org)
- A Telegram channel where the bot is an administrator, public or private
- Optional: a local Ollama server for text generation

## Setup

Activate the conda environment and install dependencies:

```bash
conda activate tg-of-ai
python -m pip install -r requirements.txt
```

On macOS with Homebrew Python, avoid `python3 -m pip ...` here because it may hit an externally managed interpreter (PEP 668).

Create a `.env` file from the example:

```bash
cp .env.example .env
```

Fill in the Telegram values and any schedule settings you want to change.

## Environment Variables

- `TELEGRAM_API_ID`: Telegram API ID from my.telegram.org
- `TELEGRAM_API_HASH`: Telegram API hash
- `TELEGRAM_BOT_TOKEN`: Bot token from BotFather
- `TELEGRAM_CHANNEL`: Channel username or identifier, for example `@your_channel` or a private channel ID like `-1001234567890`
- `TIMEZONE`: IANA timezone name, for example `UTC` or `Europe/Berlin`
- `POST_TIME`: Daily post time in `HH:MM` format
- `MIN_IMAGES`: Minimum images per post batch
- `MAX_IMAGES`: Maximum images per post batch
- `OLLAMA_HOST`: Ollama host, default `http://localhost:11434`
- `OLLAMA_MODEL`: Ollama model name, default `gemma4:e4b`
- `STATE_FILE`: Path used to remember the last successful post date
- `OUTPUT_DIR`: Directory where generated images are written
- `MAX_RETRIES`: Retry count for generation or posting failures
- `DRY_RUN`: Reserved for local experimentation

## Usage

Run a single cycle and exit:

```bash
python main.py --once
```

Run a dry-run without posting to Telegram:

```bash
python main.py --dry-run --once
```

Run continuously on the daily schedule:

```bash
python main.py
```

## Notes

- The bot must be allowed to post in the target channel.
- For private channels, set `TELEGRAM_CHANNEL` to the channel ID, not a public username.
- Dry-run mode still generates images and text, but it does not send anything to Telegram.
- If Ollama is unavailable, the bot falls back to simple generated text so the run can continue.
- Generated images are written under `OUTPUT_DIR` and reused for the posting step.

## File Layout

- `main.py`: CLI entrypoint and runtime wiring
- `config.py`: `.env` loading and validation
- `scheduler.py`: daily execution and retry handling
- `content_generator.py`: text generation using Ollama with fallback text
- `image_generator.py`: placeholder image generation
- `telegram_client.py`: Telethon setup and posting
- `models.py`: shared data structures
